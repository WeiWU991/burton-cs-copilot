## PAGE 5
### 客服的职能是什么？

> **[图片描述]:** 一位身穿绿色上衣、红色裤子的滑雪者正在滑雪。右侧为三个蓝色的职能说明框。

**客服的职能：**

1.  **专业的知识和正确心态：**
    服务意识，职业态度，情绪管理。并将以上特点运用于各类来访客户接待。

2.  **售前接待：**
    接待来访客户，针对客户售前问题进行专业解答，并进行探寻推荐，促成交易。

3.  **售中接待：**
    接待来访客户，处理客户下单后的订单查询，物流查询，交易成功前的订单问题。

4.  **售后接待：**
    接待来访客户，处理客户下单后签收后的退换货，商品问题，协助买家处理签收商品后事宜。

## PAGE 13
### 常见购物问题（退款方式）

**流程：**
付款 -> 使用支付账户 -> 申请退款 -> 等待商家审核 -> 审核退款 -> 商家审核同意退款款项转回支付账户

**支付方式选择决定了客户收到退款的方式：**

1.  **余额支付退回：** 客户使用正常余额支付（如支付宝余额、银行卡余额），退款后退回至账户余额。
2.  **信用支付退回：** 客户使用银行卡、花呗等信用支付，退款后退回至相应信用支付账户。
3.  **组合支付退回：** 订单会出现不同付款方式的搭配，如一笔订单出现余额+平台金豆+店铺优惠券的付款方式，那么退款后余额正常返还，京东豆有效期内正常返还京东，店铺优惠券有效期内退回优惠券。

## PAGE 14
### 常见购物问题（评价服务）

**多重评价：** 我们需要认知到评价对店铺的影响，对商品链接的影响，以及对客服自身的绩效判断影响。

1.  **店铺评分**
    * 确认收货后可对商品宝贝、商家服务、物流发货三个维度进行打分，分数计入整个店铺运营角度数据考核，但不对外展示。

2.  **商品评价**
    * 单独商品评价可通过文字、图片、视频方式进行商品评价，评价展示后半年内有效评价将展示在商品链接买家秀及店铺晒单区域。

3.  **客服评价**
    * 客户与客服对话时可进行的服务点评。此类指标为客服服务指标重要判断标准。

## PAGE 23
### 售前（优惠券）

**优惠券分类及叠加规则：**
店铺满减优惠券、分享券、会员券、关注券为店铺优惠券；品类券为平台类目活动报名参与。

1.  **店铺满减：** 店铺满减优惠券可与其它店铺优惠券叠加。
    * *注：店铺优惠券叠加门槛不做叠加。且为多件商品系统圈选，未达门槛订单凑单后不再圈选。*
2.  **品类券：** 品类券为报名平台类目活动，可在店铺券外额外计算。
3.  **商品券：** 个别商品专属优惠券可与其它店铺优惠券叠加。
4.  **会员券：** 店铺会员专属优惠券可与其它店铺优惠券叠加。
5.  **关注券：** 关注店铺专属优惠券可与其它店铺优惠券叠加。
6.  **分享券：** 达到分享条件可领取，可与其它店铺优惠券叠加。

## PAGE 25
### 售前（购物金）

**定义：** 购物金是挖掘品牌高价值人群并锁定未来消费，为顾客提供一站式便捷购买体验，有效拉升复购、客单！

**充值购物金 -> 使用 & 退款 -> 订单完结**

1.  **本金与权益金：** 购物金分为本金和权益金两个方面（例：设置充1,000送100（充1000得1100）购物金，则本金填写1,000，权益金填写100）。
2.  **适用范围：** 购物金适用于参与商品购买、全网优惠叠加。
    * *不适用于：* 兑换卡、电子凭证、婴儿一段奶粉、不支持抵扣定金、运费、运费险、有价服务费、直播返现、一淘返现等。
3.  **购物金退款：**
    * 买家购买购物金有365天时效，过期后未消耗的本金会自动退还。
    * 未消耗的买家购物金需求退款，可在“已卖出宝贝”为买家打开售后渠道后进行申请退款。
    * **退款金额：** 仅退还对应本金。（例：买家购买购物金充1,000送100（充1000得1100），则本金为1,000。若买家申请退款金额1100元，实际收到退款为本金1000元。）

## PAGE 26
### 售前（送礼物）

> **[图片描述]:** 流程图展示了“礼物购”的完整流程，分为付款人、收货人、订单及商家视角。

1.  **付款人：**
    * 挑选礼物 -> 拍下付款。
    * 自订单付款开始，24小时超时未领取自动退款给付款人；未满24小时内，付款人将无法主动发起退款。
    * 发送链接给朋友（记得发送安全口令）。
    * *状态分支：*
        * 超时未领取自动退款，钱款原路退回。
        * 退款成功后，钱款退回付款人。

2.  **收货人：**
    * 收到礼物链接 -> 打开淘宝输入安全口令 -> 确认地址收下礼物。
    * 发起退款 OR 确认收货 -> 收货人可评价。

3.  **订单及商家视角：**
    * 生成订单 -> 商家此时等待收货人确认收货地址或订单超时取消。
    * 订单生成地址 -> 商家操作发货。
    * 交易成功或交易关闭。
    * **商家应发货时效：** 订单创建时间 + 24小时等待期 + 订单设置的发货时效。

## PAGE 27
### 售中（常见问题）

1.  **我要现在就发货**
    * 发货时间以平台规则内为准，除特殊订单都会按照平台规则发出。

2.  **给我礼品 & 装盒**
    * 赠品参与活动条件以通知为准，不可擅自承诺及赠送。无礼品盒。

3.  **改号码改配色**
    * 均不支持修改。

4.  **帮我改价优惠点**
    * 商品价格以页面的折扣力度为准，客服没有专属优惠。

> **处理技巧：** 简要说明原因、注意回复技巧、提供次要方案、及时安抚客户情绪。

1.  **订单疑惑**

### 售中（物流异常）

> **[图片描述]:** 详细列出了5种物流异常情况的处理方式。

物流在派送期间会出现各种各样的问题，我们需要充分了解每个物流节点问题的辨别以及处理方式。

1.  **超时揽收：** 揽收超48小时未有走件信息。
    * *处理：* 需及时对接物流反馈，并安抚买家情绪告知反馈等待结果。
2.  **物流停滞：** 物流超48小时未有更新走件信息。
    * *处理：* 需先核实是否物流由集运仓，如集运影响安抚告知买家。如未有集运影响原因，对接物流反馈。并安抚买家情绪告知反馈等待结果。
3.  **走件异常：** 物流走件大致方向错误（如买家北京收货地址-苏州发货，物流由苏州发往深圳）。
    * *处理：* 反馈对接物流并安抚买家情绪告知反馈等待结果。
4.  **派件异常：** 物流显示因收方原因导致再次约定派送时间等，买家反馈未收到通知和联系。
    * *处理：* 安抚买家情绪并告知我们会尽快联系物流核对情况，并联系物流进行详细原因核对和满足买家收件需求约定派送。
5.  **实际未收：** 如物流显示签收，买家反馈包裹并未收到。
    * *处理：* 首先核对包裹签收是否为门卫/前台/丰巢签收建议买家进行查找。如确实无其他代签收情况，反馈物流查询核对，安抚买家情绪，告知尽快核对。
    * *提问：那如果签收包裹破损呢？*

### 售后（运费险）

**如何判断是否有运费险 & 应对：**

#### 1. 天猫
* **下单前：**
    * 核对商品服务保障中是否含有运费险。

#### 2. 京东
* 核对商品服务保障中是否含有免费上门退换。
* **有：** 发送赠送运费险话术。
* **无：** 告知目前无运费险活动，并探寻顾虑尝试打消。

#### 3. 抖音
* **下单前：**
    * 核对商品服务保障中是否含有运费险。
    * **有：** 发送赠送运费险话术。
    * **无：** 告知目前无运费险活动，并探寻顾虑尝试打消。
* **下单后：**
    * 订单管理中显示运费险。
    * **有：** 发送运费险使用话术，告知此单有运费险。

> * **京东：** 使用京东上门抵扣，或垫付保险公司核对是否理赔。
> * **注：** 商家赠送运费险根据客户账号信誉保险公司评估是否承保，无法保证百分百承保。

## PAGE 37
### 售后（店铺退差）

**店铺客服预案退差流程：**

1.  **有价保：**
    * 建议客户在价保有效期内申请价保。
    * **价保无法退差 (或部分退差)：** 参考价保退差预案，务必剔除价保已退金额。
    * **价保已完全退差：** 退差服务完结。

2.  **无价保：**
    * **核对客户签收是否为7天内：**
        * **7天外：** 按照店铺无法退差安抚客户情绪。
        * **7天内：** 核对客户购买时间交易快照是否含有本次活动预热。
            * **有本次活动预热：** 按照店铺无法退差安抚客户情绪。
            * **无本次活动预热：** 是否为限时或特殊折扣（折上折、限时秒杀）。
                * **有：** 告知客户限时折扣无法退差 -> 有投诉意向反馈组长。
                * **无：** BURTON 实付价 - 客户新单价 = 到手价（进行退差）。

> **退差注意事项：**
> 1.  退差金额计算后需与买家确认，买家确认无误后才可进行相应退差。
> 2.  退差金额不含个人平台权益（淘金币、无门槛红包、一淘返现、京豆等）。
> 3.  **天猫退差方式：** 买家确认收货后线上申请仅退款-退运费-修改对应差价金额。
> 4.  **京东 & 抖音：** 确认后平台小额打款至买家平台钱包。
> 5.  因退差方式原因，后续退货需将客户除货款外金额收回。

    * 确认是否满足线下退运费要求（1.线上无法申请退款 2.运费商家原因未全退）。
    * **未满足：** 可线上申请走线上申请，运费部分原因不需再次打款。
    * **满足：**
        * **抖音 & 京东：** 与客户确认运费金额，通过平台小额打款至买家APP钱包。
        * **天猫：** 与客户确认运费金额，收集客户支付宝实名账号。

## PAGE 44
### 售后（催处理）

**处理流程：**

1.  **仅退款：**
    * **商品是否发出？**
        * **未发出：** 安抚解释，会尽快核实无误处理。
        * **已发出：**
            * 未签收参考物流拦截。
            * 已签收参考无理由退换货。

2.  **审核类（退货退款 / 换货）：**
    * 是否符合店铺退换货售后预案，申请售后是否相符。
    * **超出预案：** 说明无法处理原因。
    * **在预案内：**
        * **无理由：** 系统审核，安抚顾客情绪。
        * **质量问题：** 查看是否提供凭证。
            * 确认质量问题 -> 质量问题流程处理。
            * 确认非质量问题 -> 参考无理由退换货预案。

3.  **退换类：**
    * 确认退换货包裹寄回仓库已签收。
    * **未签收：** 说明退换货需仓库验收无误后处理，可根据后台显示节点给出客户建议。
    * **已签收（核对签收时间）：**
        * **≥24H：** 说明退换货正常时效一般为物流签收后48H内尽快验收 -> 反馈仓库催促验收 -> 安抚客户情绪，主动告知为其催促加急。
        * **≤24H：** 说明退换货正常时效一般为物流签收后48H内尽快验收，根据客户情绪安抚，主动告知为其催促。
    * *注：物流显示签收阶段不要和客户保证仓库收到包裹，仅告知物流显示签收如收到仓储会尽快验收。*

4.  **发货类：**
    * 核对订单付款时间。
    * **发货时效内：** 解释正常发货时效，根据客户情绪安抚，主动告知为其催促。
    * **发货时效外：** 如订单无异常及时安抚客户情绪，并催促仓储尽快发出。
    * **特殊备注：** 如订单备注缺货或其他特殊备注，咨询售后回复方式。

## PAGE 45
### 售后（发票）

**发票类型及处理：**

1.  **电子票：**
    * 核对订单签收时间 ≤ 90天。
    * **是：** 平台申请，客户自行填写信息 -> 告知申请后会尽快处理。
    * **否：** 确认开具发票信息（抬头、税号、邮箱） -> 插旗备注，登记表格。

2.  **纸质发票：**
    * 核对订单，告知电子发票有效性。
    * **可用电子：** 参考电子票流程。
    * **要求纸质（态度强硬）：** 索要开具发票信息（抬头、税号、邮箱），不要做出承诺 -> 反馈售后确认结果为准。

3.  **专票：**
    * 核对订单，告知电子普票发票有效性。
    * **可用电子：** 参考电子票流程。
    * **要求专票（态度强硬）：** 反馈售后核对专票信息。

> **注：**
> 1.  普票开具信息购买方信息由客户提供，商家信息及开票信息中的金额、商品属性等均无法按照客户要求修改。
> 2.  个人发票可不提供购买方信息，如客户提供仅需姓名即可。
> 3.  如发送纸质发票，运费由商家承担。
> 4.  客户同账号多笔订单可合并开票。
